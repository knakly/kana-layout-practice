# 自作かな配列 練習サイト

`best_layout_char_to_position.tsv` を使った、日本語IMEに依存しない静的なかな配列練習サイトです。`d` と `k` を前置シフトキーとして扱います。現在の配列は濁音・半濁音も独立文字として配置しています。

## 使い方

1. `index.html` をブラウザで開きます。
2. 表示された練習文に合わせて物理キーを押します。
3. `次の練習文` で文を切り替え、`リスタート` で同じ文を最初から練習できます。
4. 任意の文章を貼り付けて `この文章で開始` を押すと、その文章で練習できます。

Backspace で直前の入力を戻せます。Escape は現在の前置シフト状態を解除します。

## 配列TSVの形式

`layouts/best_layout_char_to_position.tsv` は2列のTSVです。

```text
文字	配置位置
は	v
ば	dv
ぱ	dr
```

`配置位置` は物理キー列です。`v` は V キー単打、`dr` は D キーを前置シフトとして押したあと R キーを押す、という意味です。

## 前置シフトの仕様

`d` と `k` は前置シフトキーです。`d` または `k` を押した時点では文字を出力せず、次のキーと組み合わせて1文字を決定します。

例:

```text
k -> , = ぅ
d -> j = を
k -> j = つ
k -> d = あ
```

前置シフト後に未定義のキーを押した場合はミスとして扱い、シフト状態を解除します。

## 濁音/半濁音の仕様

入力エンジンは濁点 `゛` と半濁点 `゜` が配列TSVにある場合の後置修飾にも対応しています。ただし、現在の `buna1.2rc3` 系配列では、濁音・半濁音は `が`、`ば`、`ぱ` のように独立文字として直接配置されています。

```text
か + ゛ = が
は + ゛ = ば
は + ゜ = ぱ
```

練習文に `が` や `ぱ` が含まれる場合、現在の配列では配列TSVにある直接配置を優先して判定します。配列TSVに直接配置がない文字だけ、後置修飾へ分解します。

## ローカルでの起動方法

ビルドは不要です。`index.html` を直接開けば動きます。

ローカルサーバーで確認したい場合は、このフォルダで次のように起動できます。

```sh
python -m http.server 8000
```

その後、`http://localhost:8000/` を開いてください。

## GitHub Pagesでの公開方法

1. このフォルダの中身をGitHubリポジトリに置きます。
2. GitHubのリポジトリ設定で Pages を開きます。
3. 公開元として対象ブランチとルートフォルダを選択します。
4. 発行されたURLを開きます。

サーバー側処理はないため、GitHub Pagesの静的ホスティングだけで動作します。

## 別のかな配列TSVを追加する方法

1. `layouts/` に同じ2列形式のTSVを追加します。
2. `main.js` の `DEFAULT_LAYOUT_TSV` と `LAYOUT_PATH` を差し替えるか、読み込み先を追加したTSVに変更します。
3. `文字` と `配置位置` が対応していれば、前置シフトと仮想キーボード表示に反映されます。

直接 `index.html` を開いた場合はブラウザの制限でTSVを `fetch` できないことがあるため、現在の配列は `main.js` にもフォールバックとして埋め込んであります。
